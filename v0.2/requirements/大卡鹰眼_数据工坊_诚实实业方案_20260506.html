大卡鹰眼 - 数据工坊需求方案（诚实实业项目）

一、产品定位

数据工坊 = 原"固定模板"升级

从单一分析工具扩展为三类能力集合：

  • 分析模板
  • 数据下载
  • 数据解密

    • ---

    二、能力权限设计

    核心原则

    不单独授权，按数据权限自动展示

    用户有对应数据权限 → 自动看到相关能力

    三类能力

    能力绑定数据有数据权限时展示

    |------|---------|----------------|

    分析模板虚拟运单数据跨区域/同区域货物流向分析 数据下载车辆静态数据新增车辆/保有量/迁出统计下载 数据解密企业名称加密库企业名称解密入口

    新增车辆统计

    固定字段：

  • 序号、新增日期、省份、城市、区县
  • 客户名称（加密码，需解密查看）
  • 厂家、品系、车型分类、燃料种类
  • 马力、燃料分类、驱动形式、缸数、充换电
  • 新增入网数量

    • 业务规则：

  • 品系 = 品牌 + 型号
  • 相同新增日期按新增入网数量排序
  • 客户名称使用加密码，导出时不可见，需通过数据解密功能解密

    • 筛选条件（对话框逐个确认）：

  • 开始日期、结束日期
  • 省份、城市、区县
  • 厂家、品系、车型分类
  • 燃料种类、缸数、充换电

    • 保有量统计

    固定字段：

  • 序号、初登日期、省份、城市、区县
  • 客户名称（加密码，需解密查看）
  • 厂家、品系、车型分类、燃料种类
  • 马力、保有量
  • 操作（详情）

    • 筛选条件（对话框逐个确认）：

  • 初登日期范围（开始日期、结束日期）
  • 省份、城市、区县
  • 厂家、品系
  • 车型分类、燃料种类
  • 马力范围（起、止）

    • 业务规则：

  • 品系 = 品牌 + 型号
  • 按保有量排序
  • 客户名称使用加密码，导出时不可见，需通过数据解密功能解密

    • 迁出统计

    固定字段：

  • 序号、迁出日期、省份、城市、区县
  • 迁出客户名称（加密码，需解密查看）
  • 迁入客户名称（加密码，需解密查看）
  • 厂家、品系、车型分类
  • 燃料分类、燃料种类、马力
  • 驱动形式、缸数、充换电
  • 迁出车辆数

    • 筛选条件（对话框逐个确认）：

  • 迁出日期范围（开始日期、结束日期）
  • 省份、城市、区县
  • 车型分类、缸数
  • 燃料分类、燃料种类、充换电

    • 业务规则：

  • 品系 = 品牌 + 型号
  • 相同迁出日期按迁出车辆数排序
  • 迁出/迁入客户名称均使用加密码，导出时不可见，需通过数据解密功能解密

    四、数据解密流程

    完整流程

    ┌─────────────────────────────────────────────────────────┐
│                                                         │
│  [用户进入数据工坊] → [弹窗：必选能力类型，无法取消]        │
│       ↓                                                 │
│  用户选择【数据解密】                                     │
│       ↓                                                 │
│  您好，您已选择了【数据解密】                               │
│                                                         │
│  ⚠️ 本功能一次只能解密一个企业名称                         │
│  请逐个发送加密码，每次发送一个                            │
│       ↓                                                 │
│  ┌─────────────────────────────────────────────────┐     │
│  │                                                  │     │
│  │  【持续交互循环，输入框不禁锢】                     │     │
│  │                                                  │     │
│  │  用户发送：ENC_abc123                             │     │
│  │       ↓                                          │     │
│  │  系统验证：格式正确 + 数据库匹配                   │     │
│  │       ↓                                          │     │
│  │  ✅ 返回：企业名称 = 河北诚实实业集团                │     │
│  │       ↓                                          │     │
│  │  用户继续发送：ENC_def456                          │     │
│  │       ↓                                          │     │
│  │  ✅ 返回：企业名称 = 某某物流公司                    │     │
│  │       ↓                                          │     │
│  │  用户发送多个：ENC_abc123 ENC_def456               │     │
│  │       ↓                                          │     │
│  │  ❌ 提示：每次只能解密一个，请重新逐个发送            │     │
│  │       ↓                                          │     │
│  │  用户发送无关内容："今天天气不错"                    │     │
│  │       ↓                                          │     │
│  │  ❌ 提示：请输入加密码格式，示例：ENC_xxx             │     │
│  │                                                  │     │
│  │  【循环持续，直到用户主动退出或切换其他能力】          │     │
│  │                                                  │     │
│  └─────────────────────────────────────────────────┘     │
│                                                         │
│  用户主动退出 / 切换其他能力 → 结束解密会话                │
│                                                         │
└─────────────────────────────────────────────────────────┘

    规则

  • **一次只能解密一个**：用户发一个解一个，发多个则提示错误
  • **输入框不禁锢**：可持续交互，无需重复选择"数据解密"能力
  • **只能发加密码**：发其他内容提示格式错误
  • **无次数限制**：可连续解密多个，逐个发送即可
  • **退出方式**：用户主动退出或切换其他能力

    七、诚实实业 - 模板沉淀计划（待与客户沟通）

    目标

    将诚实实业的分析需求沉淀为系统内置模板，后续有相同数据权限的客户可直接使用。

    沉淀内容

    #模板名分析方向使用数据 :---:---:---:--- 1新增车辆分布分析按地区、品牌、车型、燃料类型统计新增车辆数量和结构分布新增车辆 2品牌保有量结构分析按厂家、车型、燃料类型统计当前保有量占比和区域分布保有量 3物流企业迁出分析按迁出企业、迁出地区统计车辆迁出数量和流向迁出车辆 4新能源车辆增长分析对比新能源与传统燃料车辆的新增量和保有量占比新增 + 保有量 5区域车辆变动分析结合保有量分布和迁出流向，看车辆在各区域间的流动情况保有量 + 迁出 6车辆数据综合分析综合新增、保有、迁出三项数据，看车辆数据的整体分布和变化情况新增 + 保有量 + 迁出

    沉淀流程

    诚实实业提出需求 → 产品设计模板结构 → 研发实现 → 测试验证 → 纳入数据工坊
