大卡鹰眼 — 模板化查询 PRD v1

1. 概述

1.1 背景

当前大卡鹰眼采用"自由输入 → NL2SQL"模式，用户自然语言提问后直接查数出报告。但存在两个问题：

  • 用户不知道能问什么，产生"空输入"或"胡问"
  • 用户输入的信息不全，系统需要反复追问

    • 本PRD提出模板化查询模式：用户先选模板，系统自动问参数，填完就出结果。

    1.2 核心原则

  • **模板必选** — 没有自由输入，只能选模板
  • **对话式填参** — 系统问，用户答，像聊天一样
  • **容错机制** — 答错了最多追问2次，第3次走默认值
  • **一次出清** — 结果输出后关闭对话框，不再接受输入

    • ---

    2. 用户流程

    用户进入大卡鹰眼
     ↓
展示模板列表（8个模板，每个一句话说明）
     ↓
用户选择模板（点击或输入编号）
     ↓
系统自动识别模板 → 解析所需参数
     ↓
逐一向用户询问参数（最多3个参数）
   ↓   ↓   ↓
用户回答 → 校验格式 → 通过→下一参数
                  → 不通过 → 追问（最多2次）
                            → 仍不通过 → 用默认值
     ↓
所有参数收集完毕
     ↓
生成查询 → 输出结果（报告/图表/表格）
     ↓
结果展示 → 对话框关闭（不可再输入）

    4. 参数问答机制

    4.1 询问方式

    选定模板后，系统根据模板参数的`required_params`列表，逐一询问。

    询问话术模板：

    您好，您选择了「{模板名称}」模板，需要确认以下几个参数：

1️⃣ 时间范围（示例：2026年3月 / 上月 / 今年一季度）
2️⃣ 城市（示例：成都 / 宜宾）
3️⃣ 货类（示例：粮食 / 煤炭 / 全部）

请先告诉我【时间范围】：

    4.2 每次只问一个参数

    系统：请先告诉我【时间范围】：
用户：上月
系统：好的，时间范围确认为「上月」。
      接下来请告诉我【城市】：
用户：成都
系统：好的，城市确认为「成都」。
      最后一个问题，请告诉我【货类】（不填则默认"全部"）：
用户：粮食
系统：好的，货类确认为「粮食」。
      正在为您查询，请稍候...

    4.3 校验规则

    每个参数关联一个校验函数：

    参数校验规则失败提示

    |------|---------|---------|

    时间范围匹配"YYYY年MM月" / "上月" / "今年Q1"等"时间格式不正确，请参考示例：2026年3月 / 上月 / 今年一季度" 城市匹配城市名称（可模糊匹配）"未识别到城市名称，请重新输入，例如：成都、宜宾" 收货城市同"城市"，可选填"全部""未识别到城市名称，请输入城市名或填写「全部」" 货类匹配17个一级货类之一"未识别到货类，请输入一级货类名称，例如：粮食、煤炭" 企业名称非空即可"企业名称不能为空" 城市A/城市B同"城市"，两个城市不能相同"城市A和城市B不能相同，请重新输入"

    4.4 容错机制

    第1次：用户输入格式错误
  → 系统提示错误原因 + 示例 + 重新询问

第2次：用户再次输入错误
  → 系统再次提示 + 说明"再错将使用默认值"

第3次：用户第三次输入错误或未输入
  → 系统直接使用默认值，进入下一参数

示例对话：

系统：请告诉我【城市】：
用户：我不确定
系统：未识别到城市名称，请重新输入，例如：成都、宜宾
用户：随便吧
系统：城市识别失败，将使用默认值「成都」
      接下来请告诉我【货类】：

    5. 输出与关闭

    5.1 结果输出

    所有参数收集完毕后，系统执行：

  • 模板参数填充 → 生成完整查询语句
  • 调用NL2SQL → 查数据
  • 输出结果（报告/图表/数据表格）

    • 输出格式示例：

    🔍 查询完成 | 货物流向分析 | 成都 → 全国 | 2026年3月

【流向总览】
从成都发往全国的货物中，前三流向城市为：
1. 重庆 — 23.5%
2. 宜宾 — 15.2%
3. 西安 — 8.7%

【货类分布】
- 粮食类：42.3%
- 煤炭类：18.7%
- 建材类：15.1%

【查看完整数据...】

    5.2 对话关闭

    结果输出后，对话框进入只读状态：

  • 用户不能再输入新的查询
  • 显示提示："如需新的查询，请重新发起对话"
  • 如需继续分析，用户需回到模板选择页重新开始

    8. 技术实现要点

  • **模板定义** — JSON配置化，可随时增删模板
  • **参数校验** — 规则引擎，按参数类型注册校验函数
  • **对话状态机** — 4个状态：TEMPLATE_SELECT → PARAM_INPUT → WAITING_RESULT → COMPLETED
  • **超时处理** — 60秒无响应→自动走默认值继续
  • **日志记录** — 每次查询记录：模板ID、参数值、是否用了默认值、用户重试次数
