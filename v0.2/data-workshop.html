数据工坊产品功能方案

诚实实业项目 | 2026-05-06

一、产品定位

数据工坊 = 原"固定模板"升级，从单一分析工具扩展为三类能力集合：

  • 分析模板
  • 数据下载
  • 数据解密

二、能力权限设计

核心原则：不单独授权，按数据权限自动展示。

能力分析模板
绑定数据虚拟运单数据
有权限时展示跨区域/同区域货物流向分析

三、数据下载流程

[用户进入数据工坊] → [弹窗：必选能力类型，无法取消]
    ↓
用户选择【数据下载】
    ↓
您好，您已选择了【数据下载】
请从以下数据中选择要下载的内容：

① 新增车辆统计
② 保有量统计
③ 迁出统计
    ↓
用户选择【新增车辆统计】
    ↓
您选择了【新增车辆统计】，输出字段如下：
（固定字段，不可更改）

序号、新增日期、省份、城市、区县、客户名称（掩码）
厂家、品系、车型分类、燃料种类、马力
燃料分类、驱动形式、缸数、充换电、新增入网数量
    ↓
请对以下条件做填充：

① 开始日期    格式：YYYY-MM-DD    示例：2026-03-01
② 结束日期    格式：YYYY-MM-DD    示例：2026-04-01
③ 省份        可填：省份名称       默认：河北省
④ 城市        可填：城市名称/全部   示例：石家庄、全部
⑤ 区县        可填：区县名称/全部   示例：宁晋县、全部
⑥ 厂家        可填：厂家名称/不限   示例：一汽长春、不限
⑦ 品系        可填：品牌+型号/不限  示例：德龙L3000、不限
⑧ 车型分类    可填：载货/牵引/自卸等/不限
⑨ 燃料种类    可填：柴油/天然气/混合动力等/不限
⑩ 缸数        可填：四缸/六缸等/不限
⑪ 充换电      可填：充电/换电/不限
    ↓
┌────────────────用户回答─────────────────┐
│ 可能填1个："开始日期2026-03-01"          │
│ 可能填2个："开始日期3月1日，结束日期4月1日"│
│ 可能全填：所有条件一起说                   │
│ 可能填错："时间昨天"、"城市火星"          │
│ 可能说无关话："今天天气不错"              │
└──────────────────────────────────────────┘
    ↓
┌──── 系统自动识别 ────┐
│ 1. 提取：哪些条件被填了？ │
│ 2. 校验：填了的对不对？   │
│ 3. 纠正：错的/无关的逐个处理│
└──────────────────────┘
    ↓
分情况处理：
  ├─ 全部填对 → 已为您确认所有条件，正在生成下载链接…
  ├─ 填了一部分且全对 → 好的，开始日期确认为2026-03-01，
  │                      结束日期确认为2026-04-01
  │                     还差省份、城市等，请继续：
  │                     → 只列剩余条件，已确认的不再问
  ├─ 部分填错 → 好的开始日期没问题
  │             城市「火星」无法识别，
  │             请参考示例：石家庄、唐山、全部
  │             → 只纠正错的，对的不动
  └─ 回答无关 → 抱歉，我需要您填写的是筛选条件哦
               追问一次 → 仍无关 → 走默认值继续
    ↓
全部条件确认 → 正在为您生成CSV文件…
    ↓
[对话框生成【下载CSV】按钮]
    ↓
用户点击下载 → 输出CSV文件
    ↓
[输入框禁锢，不可再输入]
本次下载已完成，如需新查询请重新发起会话

四、数据解密流程

[用户进入数据工坊] → [弹窗：必选能力类型，无法取消]
    ↓
用户选择【数据解密】
    ↓
您好，您已选择了【数据解密】

⚠️ 本功能一次只能解密一个企业名称
请逐个发送加密码，每次发送一个
    ↓
┌─────────────────────────────────────────────────┐
│ 【持续交互循环，输入框不禁锢】                     │
│                                                  │
│ 用户发送：ENC_abc123                              │
│      ↓                                           │
│ 系统验证：格式正确 + 数据库匹配                    │
│      ↓                                           │
│ ✅ 返回：企业名称 = 河北诚实实业集团                │
│      ↓                                           │
│ 用户继续发送：ENC_def456                           │
│      ↓                                           │
│ ✅ 返回：企业名称 = 某某物流公司                    │
│      ↓                                           │
│ 用户发送多个：ENC_abc123 ENC_def456                │
│      ↓                                           │
│ ❌ 提示：每次只能解密一个，请重新逐个发送            │
│      ↓                                           │
│ 用户发送无关内容："今天天气不错"                    │
│      ↓                                           │
│ ❌ 提示：请输入加密码格式，示例：ENC_xxx             │
│                                                  │
│ 【循环持续，直到用户主动退出或切换其他能力】          │
└─────────────────────────────────────────────────┘
    ↓
用户主动退出 / 切换其他能力 → 结束解密会话
