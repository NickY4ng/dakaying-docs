固定模板深度分析报告 需求文档

版本：v1.0 | 日期：2026-04-29 | 编写：杨航

一、要做什么

从多轮对话+自由输入，转向固定模板深度分析报告。

用户必选模板 → 一次性看到所有条件 → 边聊边填 → 填完出报告 → 对话结束。

二、完整流程

[用户进入] → [弹窗：必选模板，无法取消，标题提示语]
    ↓
用户选了【货物流向分析】
    ↓
您好，您已选择了【货物流向分析】模板，
请对以下条件做填充：

① 时间范围    格式：哪年哪月~哪年哪月
              数据范围：2025年12月 ~ 2026年3月
              示例：2026年1月~2026年3月

② 出发地      可填：省份 / 城市 / 区县
              可多选，可不限
              示例：四川、成都、双流

③ 到达地      可填：省份 / 城市 / 区县
              可多选，可不限
              示例：重庆
              ⚠️ 出发地和到达地不可都不限

④ 货类        可多选，可不限
              枚举：粮食 煤炭 钢铁 建材 化工
                    机械设备 电子电器 日用百货
                    食品饮料 医药 纺织 快递
                    物流 垃圾 渣土 砂石 其他
    ↓
┌────────────────用户回答─────────────────┐
│ 可能填1个："时间2026年1月到3月"          │
│ 可能填2个："时间2月到3月，出发地成都"    │
│ 可能全填：4个条件一起说                  │
│ 可能填错："时间昨天"、"出发地火星"       │
│ 可能说无关话："今天天气不错"             │
└──────────────────────────────────────────┘
    ↓
┌──── 系统自动识别 ────┐
│ 1. 提取：哪些条件被填了？ │
│ 2. 校验：填了的对不对？   │
│ 3. 纠正：错的/无关的逐个处理│
└──────────────────────┘
    ↓
分情况处理：
  ├─ 全部填对 → 已为您确认所有条件，正在生成报告…
  ├─ 填了一部分且全对 → 好的，时间确认为2月~3月，
  │                      出发地确认为成都
  │                     还差到达地和货类，请继续：
  │                     → 只列剩余条件，已确认的不再问
  ├─ 部分填错 → 好的时间没问题
  │             出发地「火星」无法识别，
  │             请参考示例：成都、宜宾、四川
  │             → 只纠正错的，对的不动
  └─ 回答无关 → 抱歉，我需要您填写的是分析条件哦
               追问一次 → 仍无关 → 走默认值继续
    ↓
全部条件确认 → 正在为您生成深度分析报告…
    ↓
[输出深度分析报告]
    ↓
[输入框禁锢，不可再输入]
本次分析已完成，如需新查询请重新发起会话

三、对话示例

场景1：用户一次填一个

      ① 时间范围 — 哪年哪月~哪年哪月（数据范围：2025年12月~2026年3月）

      ③ 到达地 — 省份/城市/区县，可多选可不限（不可都不限）
      ④ 货类 — 可多选可不限（共17类可选）

用户：时间就2026年1月到3月吧

系统：好的，时间确认为2026年1月~3月 ✓
      还差3个条件：

      ② 出发地 — 示例：四川、成都
      ③ 到达地 — 示例：重庆
      ④ 货类 — 共17类可选

      请继续

用户：出发地成都，到达地重庆

      还差1个条件：

      ④ 货类 — 可选：粮食 煤炭 钢铁 建材 化工 机械设备
                     电子电器 日用百货 食品饮料 医药 纺织
                     快递 物流 垃圾 渣土 砂石 其他

用户：煤炭和建材

系统：好的，货类确认为煤炭、建材 ✓

      全部条件已确认：
      - 时间：2026年1月~3月
      - 出发地：成都
      - 到达地：重庆
      - 货类：煤炭、建材

      正在为您生成深度分析报告…

场景2：用户一次填完，但部分填错

系统：好的，货类确认为粮食 ✓，到达地确认为重庆 ✓

      全部条件已确认：
      - 时间：2026年1月~3月
      - 出发地：成都

四、前端要求

  • 开启新会话即弹出模板选择弹窗，弹窗标题：「请选择模板，填写条件，开启深度分析」；下掉自定义模板功能；不可取消，必须选择一个模板才能进入对话
  • 报告输出后输入框强制禁锢，显示「本次分析已完成，如需新查询请重新发起会话」
  • 输入框和显示区域固定高度，内容过多时内部可上下滚动，不撑大页面导致无法滑动
  • 系统模板浮窗默认显示，支持 Markdown 预览
  • 模板选择后去掉关闭叉号，只能重新选择/切换，不能关闭
  • 新会话不继承上次模板选择，每次重新选
  • 使用帮助按钮放到页面左下方
  • 登录手机号隐藏，放到点击人像里，中间四位掩码（如 138****1234）
